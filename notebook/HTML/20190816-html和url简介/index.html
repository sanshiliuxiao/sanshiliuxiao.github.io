    HTML 简介

    HTML： 超文本标记语言(HyperText Markup Language)，于 1990 年，由 Tim Berners-lee（李爵士）发明。 2014 年，HTML5 发布，目前广泛使用的版本就是 HTML5。

    网页的基本骨架

    <!-- <!DOCTYPE html> 会使浏览器按照 html5 的标准解析网页-->
<!DOCTYPE html>
<!-- lang 默认是 en 中文是 zh-CN -->
<html lang="en">
    <!-- head 放置网页的元信息 不会出现在网页上 -->
    <head>
    <!-- meta 网页的元数据 utf-8 表示 编码方式 -->
    <meta charset="utf-8" />
    <!-- 网页标题 -->
    <title>hello wrold</title>
    </head>
    <!-- 网页的主体  -->
    <body></body>
</html>

    网页的基本概念

    1. 标签
      HTML 代码由许许多多的标签（tags）组成。

      <!-- title 标签 -->
<title>网页标题</title>

      标签可以嵌套，但要保证正确的闭合顺序

      <!-- 正确 -->
<div><p>hello wrold</p></div>
<!-- 错误 -->
<div><p>hello wrold</div></p>

    2. 元素
      标签是从源码角度来看， 元素是从编程角度来看，它们本质上是同一种东西。浏览器渲染网页，会把源码解析成一个标签树，每一个标签就是一个节点（node），称为元素（element）。
      元素主要分为两大类：块级元素（block）、行内元素（inline）。

    3. 属性
      属性（attribute）是标签的额外信息， 因为 HTMl 大小写不敏感（都会变成小写解析），因此 onclick 和 onClick 可以被认为是同一个属性

      <!-- 
  src width 都是属性
  demo.jpg 500 都是属性值
-->
<img src="demo.jpg" width="500" />

    4. 注释
      HTML 代码可以进行注释，浏览器会自动忽略注释

      <!-- 这是一个注释 -->

<!-- 
  允许多行注释
  这是多行注释
-->

    HTML 处理空格的规则

    1. 标签内容的头部和尾部的空格，一律忽略。

      <p>hello wrold</p>
<!-- 等价 -->
<p>hello wrold</p>

    2. 标签内连续多个空格（包含 \t），合并成一个空格。

      <p>hello world</p>
<!-- 等价 -->
<p>hello wrold</p>

    3. 文本内换行符（\n）和回车符（\r），替换成空格。

      <p>hello wrold</p>
<!-- 等价 -->
<p>hello wrold</p>

    4. 标签与标签之间的换行符和回车符，同样替换成空格。

      <p>
  hello

  <span>wro</span>
  <span>ld</span>
</p>
<!-- 等价 -->
<p>hello <span>wro</span> <span>ld</span></p>

    URL 简介

    URL：统一资源定位符（Uniform Resource Locator）

    互联网中，任何一种可访问的文件，都可以理解成一种资源，一个 URL 对应一个资源，但一个资源可以具有多个 URL。 URL 是互联网的基础，最常见的形式就是网页通过链接包含其他 URL，用户点击链接，就能够从一个 URL 跳转到另一个 URL。

    网址的组成

    https://www.example.com:80/path/to/myfile.html?key1=value1&key2=value2#anchor

    注意 www.example.com 这个域名中的 www 是有历史原因的。

    参考: 为什么越来越多的网站域名不加「www」前缀？

    1. 协议（scheme）
      协议是浏览器请求服务器资源的方法，如 http、https、ftp等。如果省略协议，浏览器会自动添加上默认的协议，一般是 http。

    2. 主机（host）
      主机表示资源所在地，一般由域名进行确定，不过域名只是为了便于记忆，起核心还是 IP 地址。
      同一个域名下面可以具有多个子域名，比如 www.example.com 就是 example.com 的子域名。

      // 某个域名
www.example.com

// 指向某个 IP
181.123.123.10

    3. 端口（port）
      同一个域名下面也可以通过端口区分不同的网站，不过一般网站的默认端口是 80， 因此 URL 可以忽略这个参数。

      www.example.com:80

// 默认等价
www.example.com

    4. 路径（path）
      资源所在网站的位置，路径可能只包含文件名，服务器默认情况下回跳转到该目录下的 index.html 文件，或者列出这个目录下的所有文件，至于如何显示，取决于服务器的设置。

      www.example.com

// 默认等价
www.example.com/index.html

    5. 查询参数（parameter）
      查询参数是提供给服务器的额外参数，紧跟在路径后面，使用 ? 与路径分割。查询参数可以有一组或多组，通过键值对的方式进行设置，使用 = 连接键值对，多组参数使用 & 进行连接。
      查询参数不是必须的，如果不需要，则可以不出现在 URL 上。

      www.example.com/a/b/index.html?key1=value1&key2=value2

    6. 锚点（anchor）
      网页内部的定位点，通常放置在 URL 的末尾，通过 # 分割。 如果 URL 上存在锚点 浏览器加载页面后，会自动定位到元素上具有 id 属性，并且属性值等于锚点字符的位置。

      www.example.com/a/b/index.html#1234
      <!-- 省略一些内容 -->
<p id="1234">我可以被自动定位到的哦</p>

    URL 字符

    URL 的各个组成部分，只能使用以下字符

    • 26 个英文字母（大小写区分）
    • 10 个阿拉伯数字
    • 连词号（-）
    • 句点（.）
    • 下划线（_）

    另外还有 18 个保留字符，它们只能出现在特定的位置，比如 查询参数是以字符问号（？） 开头，那么问号就只能出现在查询参数开头。

    如果需要在其他地方进行使用，那么就必须使用它们的转义形式，转义的方法，就是在这些字符的 ASCLL 编码的十六进制形式，前面加上 百分号（%）。

    符号 转义 符号 转义 符号 转义
    ! %21 # %23 $ %24
    & %26 ‘ %27 ( %28
    ) %29 * %2A + %2B
    , %2C / %2F : %3A
    ; %3B = %3D ? %3F
    @ %40 [ %5B ] %5D

    值得注意的是，空格的转义形式是 %20，对于那些包含空格文件的文件名，这个转义是必须的。

    既不属于合法字符，也不属于保留字符，比如汉字。理论上是不需要手动转义，浏览器会自动将其转义，发送给服务器。转义的方法是使用这些字符的utf-8 十六进制编码，每两位算作一组，然后每组添加百分号

    www.example.com/中国.html

// 等价
www.example.com/%e4%b8%ad%e5%9b%bd.html

// 原因
中 十六进制编码是 e4b8ad，转义 %e4%b8%ad
国 十六进制编码是 e59bbd，转义 %e5%9b%bd

    相对 URL 和 绝对 URL

    绝对 URL 是只靠 URL 本身就能够确定资源的位置，而相对 URL 还必须结合当前网页的位置才能够定位资源。

    相对 URL ，如果以 / 开头，就表示相对网站的根目录，如果直接写路径，或者以 . 或 ..开头，则以当前路径为起点，推算资源的位置。

    // 一个绝对 url
https://www.example.com/path/index.html

// 相对 url

假设当前网页 url 为 https://www.example.com/a/b/x.html

/c/y.html
// 等价
https://www.example.com/c/y.html

xx.html
//等价
https://www.example.com/a/b/xx.html

./z.html
// 等价
https://www.example.com/a/b/z.html

../m.html
// 等价
https://www.example.com/a/m.html

//  . .. 还能够多个连用

../.././n.html
// 等价
https://www.example.com/n.html

    base 标签

    <base> 是一个自闭合标签，它指定网页内部所有相对 url 的计算基准，整张网页最多只能有一个 <base> 标签，必须放置 <head> 标签里面。
    注意，这个标签不是必须存在的,而且也很少用。

    https://www.example.com/a/b/c.html
    <head>
  <base href="https://www.example.com/files/" target="_blank" />
</head>

<!-- 那么相对路径的计算基准
     不是www.example.com/a/b
     而是 www.example.com/files/ 
-->
