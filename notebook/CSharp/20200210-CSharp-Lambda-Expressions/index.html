    • C# 进阶之 Lambda

    定义

    // Lambda expression
(parameters) => expression-or-statement-block
// anonymous method
delegate  (parameters) { statement-block }

    Lambda 表达式是一种可以替代委托实例的匿名方法，编译器会将 Lambda 转化成两种形式：委托实例 或 Expression 。
    一般我们会在关联委托（或事件） 时或者使用 LINQ 中时去使用 Lambda 表达式，因为这非常的方便。
    另外，Lambda 表达式不允许使用含有 ref 或 out 关键字的参数。

    () => "1"; // 无参，有返回值
(int x, int y) => x + y; // 有参 ，有返回值
(int x, int y) => Console.WriteLine(x + y); // 有参，返回 void
// 如果有多行操作，则需要有 {} 。
() => {
    Console.WriteLine("Hello");
} // 无参， 无返回值
() => {

    Console.WriteLine("Hello");
    return "World";
} // 无参，有返回值
(int x, int y) => {
    int z = 4;
    int a = x+y+z;
    return a;
} // 有参，有返回值
(int x, int y) => {
    Console.WriteLine($"total: {x +y }");
} // 有参，无返回值
// 更进一步，编译器能够进行类型推断。因此，还能这样写
(x, y) => x+y;  // 非常的简洁。

    从 C# 6.0 开始， Lambda 表达式，允许被用在普通方法和属性的声明上。这样能够有效的缩短代码行数。。。

    class Test
{
    public int CalcateSum(int a, int b)
    {
        return a+b;
    }
    // ok
    public int CalcateSum2(int a, int b) => a + b;

    //error
    public int CalcateSum3(int a, int b) => {
        return a+b;
    }
    private string name;
    public string Name
    {
        get => name;
        set => name = value;
    } // 一般很少这样使用
}

    如果关联的事件想要去取消的话，则应该使用一个变量去存储 Lambda 表达式。

    public event EventHandler ValueChanged;
public lambda = (s, e) => Console.WriteLine(e.ToString());
public AddOrRemoveEventListen()
{
    ValueChanged += lambda;
    ValueChanged -= lambda;
}
